Comandos GIT

Git clone Git add (. adiciona tudo, git reset -- NOMEARQUIVO para tirar o que não deseja adicionar, git add -i forma de adicionar interativa.) Git commit -m

Quando é feito um novo commit é criado uma nova árvore, um novo blob. Podemos usar o git log para ver o histórico (melhor de visualizar no GitHub).

Git reset (--hard e --soft) Git checkout -b feature, equivalente a gib branch feature

Para verificarmos as branchs existentes podemos usar um tree .git/refs. As branchs são etiquetas que apontam para um commit.

Git switch feature Git log Git merge feature

Mudança realizada na branch feature

Git rebase master Git branch -D NOMEDABRANCH Git commit --amend Git revert HASHDOCOMMIT

Para trabalhar com nova branch

  • Primeiro: Executar o comando git checkout -b NOMEDABRANCH, para criar uma nova branch.
  • Segundo: Fazer as alterações desejadas e após realizar o git add, git commit e git push (origin NOMEDABRANCH) na branch criada.
  • Terceiro: Mudar para a branch principal (onde deseja unir o conteúdo da branch criada) e executar o git merge NOMEDABRANCH (O merge traz o conteúdo da branch digitada para a branch atual).
  • Quarto: Na branch principal precisamos realizar o git push pois no momento que faz o merge fica apenas no repositório local.

História do GIT (12 anos atrás)

Antes era feito com Patches e Tarballs

Bitkeeper - Um dos primeiros versionadores de códigos.

Git foi desenvolvido por volta de 2015.

Diff e Patch do Linux - diff para identificar a diferença, patch que é o "remendo", que une as diferenças.

Branch - galho: duplica o diretório, e gera os patches. PullRequest é basicamente uma branch do projeto.

1°: criar branch, fazer alterações necessárias, realizar add, commit e push. 2°: no GitHub criar o PullRequest, no compare colocar a branch trabalhada atualmente, na descrição do PullRequest colocar "closed" e a issue se tiver e quiser fechar ao mergear. 3°: mergear a PullRequest no GitHub, para subir as alterações. 4°: no VSCode fazer um pull para sincronizar o repositório remoto com o repositório local.
